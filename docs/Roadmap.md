# YORO.gg Product Roadmap v1

문서 상태: 공식 제품 로드맵  
적용 범위: 6개월, 1년, 2년, 5년 제품 성장 계획

## 1. Roadmap 원칙

YORO.gg Roadmap은 기능 목록이 아니라 제품 loop를 강화하는 순서이다.

우선순위:

1. 제품 정체성 정리
2. Streamer와 Viewer 연결 강화
3. 방송 운영 안정화
4. Overlay Studio 완성
5. Community와 Tournament 확장
6. KR / JP 성장

## 2. 6개월 Roadmap

목표: YORO.gg를 하나의 제품으로 정리한다.

주요 과제:

- Product Constitution 적용
- IA v3 기준 navigation 재정리
- Today Live dashboard 설계 반영
- Overlay Studio 통합 설계
- Design System v1 foundation 확정
- Public search to streamer discovery flow 정리
- My YORO 개인화 구조 정의
- Admin과 Streamer 기능 분리

성공 기준:

- 사용자가 제품 목적을 첫 화면에서 이해한다.
- Streamer가 오늘 방송 상태를 1분 안에 판단한다.
- Overlay source 설정 flow가 명확해진다.

## 3. 1년 Roadmap

목표: Streamer와 Viewer 참여 loop를 완성한다.

주요 과제:

- Streamer Profile 강화
- 시참 신청과 대기열 experience 개선
- Viewer participation history
- Streamer Space v1
- Party Finder v1
- Tournament Host v1 안정화
- Overlay source별 preview / test 완성
- Broadcast Summary v1

성공 기준:

- 시참 신청부터 참여 완료까지 flow가 끊기지 않는다.
- 스트리머가 방송 후 성과를 확인할 수 있다.
- 커뮤니티가 게시판이 아니라 스트리머 관계 중심으로 동작한다.

## 4. 2년 Roadmap

목표: KR / JP LoL 스트리머 플랫폼으로 성장한다.

주요 과제:

- KR / JP cross-language community 강화
- Viewer reputation system
- Streamer growth analytics
- Tournament season system
- Community moderation tools
- Team / party matching quality 개선
- Creator monetization 준비

성공 기준:

- 반복 참여하는 User와 Streamer가 증가한다.
- Streamer Community가 방송 밖에서도 유지된다.
- 대회와 시참이 신규 유저 유입 loop로 작동한다.

## 5. 5년 Roadmap

목표: 참여형 게임 방송 운영 플랫폼이 된다.

주요 과제:

- League of Legends 외 참여형 게임 확장 검토
- Multi-game streamer profile
- Cross-platform broadcast tools
- Community event marketplace
- Advanced overlay personalization
- Creator ecosystem partnerships

성공 기준:

- YORO.gg가 KR / JP 참여형 방송 운영의 대표 플랫폼이 된다.
- Streamer와 Viewer의 장기 관계 데이터가 제품 핵심 자산이 된다.
- 검색, 방송, 커뮤니티, 대회가 하나의 성장 loop로 안정화된다.
